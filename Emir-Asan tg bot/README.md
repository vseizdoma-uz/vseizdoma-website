# 🤖 Architect Assistant Bot

Минималистичный Telegram-бот помощник для архитекторов.

## 📋 Описание

Бот помогает архитекторам быстро структурировать техническое задание и составлять план работы над проектом.

**Возможности:**
- ✅ Анализ технического задания
- ✅ Структурирование требований
- ✅ Формирование плана работ
- ✅ Красивый текстовый вывод
- ✅ Простое меню управления

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка токена

Получите токен у [@BotFather](https://t.me/BotFather) в Telegram.

Затем установите токен одним из способов:

**Вариант А:** Переменная окружения (рекомендуется)
```bash
# Windows
set BOT_TOKEN=ваш_токен_здесь

# Linux/Mac
export BOT_TOKEN=ваш_токен_здесь
```

**Вариант Б:** Прямо в коде
Откройте `bot/index.js` и замените:
```javascript
const TOKEN = process.env.BOT_TOKEN || 'YOUR_BOT_TOKEN_HERE';
```
на:
```javascript
const TOKEN = 'ваш_реальный_токен';
```

### 3. Запуск бота

```bash
npm start
```

Вы увидите:
```
🤖 Architect Assistant Bot запущен!
📡 Режим: polling
⏳ Ожидание сообщений...
```

## 🎯 Использование

### Команды бота

- `/start` — Приветствие и главное меню

### Кнопки меню

- **📝 Новое ТЗ** — Отправить техническое задание для анализа
- **📐 Пример анализа** — Посмотреть пример работы бота
- **ℹ️ Помощь** — Справка по использованию

### Как отправить ТЗ

Просто напишите боту описание вашего проекта:

```
Необходимо спроектировать частный дом площадью 200 м²
в современном стиле. Участок 12 соток. Требуется
4 спальни, гостиная-кухня, 3 санузла, гараж.
Важна энергоэффективность.
```

Бот проанализирует текст и выдаст структурированный ответ с:
- Основными параметрами проекта
- Ключевыми требованиями
- Планом действий
- Рекомендуемой последовательностью работ

## 🔧 Технические детали

- **Язык:** JavaScript (Node.js)
- **Библиотека:** node-telegram-bot-api
- **Режим работы:** Polling (без webhook)
- **База данных:** Не используется
- **Внешние API:** Не используются

## ⚙️ Особенности

✅ **Никакой оплаты** — полностью бесплатный
✅ **Никакой базы данных** — работает без хранения данных
✅ **Никаких внешних API** — автономная работа
✅ **Простой код** — легко понять и модифицировать
✅ **Polling режим** — запускается где угодно
